Getting started with ACEv12 - Creating a Simple Unit Test

Learn how ACE unit tests help you with rapid Test Driven Development by creating and executing a simple Java Unit Test.

Import a project

Click Import to add the required project to your workspace:

  1. The application named MyApplication will be imported into your workspace.
    • MyApplication provides a single message flow named MyMessageFlow which receives data over HTTP.
    • The HTTPInput node is configured to expect an XML message which conforms to MyModel.xsd although the HTTPInput node itself is not set up to validate the message immediately.
    • The next node in the flow is a graphical mapping node which transforms the message into a JSON output.
    • The HTTPReply node at the end of the flow returns the JSON to the requester.
  2. Now move to the Prepare tab where we describe how to create a unit test for the flow we have just imported.

Create a unit test for the mapping node in the message flow

MyApplication contains MyMessageFlow which should already be open in your workspace:

  1. Right-click the node in the message flow named Mapping and choose the menu option to Create testcase. The New Java Unit Test Class wizard will launch. Note that the Node Location fields will already have been populated with the name of the application, message flow, node and terminals.
  2. In the section labelled Select an input file to construct a message assembly to invoke the node, click the Browse button. Select the file RawInput.xml from the Other resources section of the MyApplication project in your workspace. This option creates a unit test using serialized input data which you already have available.
  3. In the section labelled Select an output file to compare against the actual output message, click the Browse button. Select the file RawOutput.json from the Other resources section of the MyApplication project in your workspace.
  4. In the section labelled Assertions and Matchers to be included in the generated test, you should see that this has caused the option to Assert output message body data to become ticked. You can scroll through the preview pane to see what the generated unit test will look like. Click the Finish button to generate the test.
  5. You will find that a Test Project named MyApplication_IntegrationTest is generated for you. The generated java unit test is opened for you (you can also find it in MyMessageFlow_Mapping_001_IntegrationTest.java). The project has a resources folder where your input and output data files will have been placed. The generated Test Project has a project reference configured which points the Test Project at the application MyApplication. On the next tab we will describe how to run the unit test you have just created.

Run the Unit Test

  1. In the Application Development view (by default in the top left of the Toolkit), locate the Test Project which you just created.
  2. Right click the Test Project named MyApplication_IntegrationTest and choose the option Run as > Integration Server JUnit Test Runner.
  3. After a few seconds, the Console view should be opened, which will show an independent integration server being started in order to execute the unit test. There is no deployment step required - This is taken care of for you so that the application under test, and the Test Project which we just created are automatically available to the server.
  4. The JUnit view will open in the top left corner (just to the right of the Application Development view and the Patterns Explorer) and will show that the test which was generated ran successfully and passed.

This tutorial has shown how to create and excute a simple Unit Test.
